# notes-to-docx

Инструмент для создания DOCX-документов из фотографий рукописных конспектов или текстовых файлов.

## Режимы работы

### Mode 1 — Простая вставка фото
Берёт фотографии из папки (отсортированные по имени) и размещает их по две на лист A4 (альбомная ориентация).

### Mode 2 — OCR + рендер шрифтом
1. Каждое фото отправляется в AI (Gemini или OpenRouter) для распознавания текста
2. AI определяет правильный порядок страниц
3. Текст рендерится пользовательским TTF-шрифтом в изображения
4. Изображения вставляются в DOCX

### Mode 3 — Печать из текстового файла

**Два формата ввода:**

| Файл | Режим | Возможности |
|------|-------|-------------|
| `print.txt` | Обычный текст | Простой текст, переносы строк |
| `print.md` | Markdown | Заголовки, центровка, разные размеры шрифта |

**Приоритет:** если есть `print.md`, он используется. Иначе — `print.txt`.

### Синтаксис markdown (`print.md`)

```markdown
# Заголовок          — крупный шрифт (1.6x), по центру
## Подзаголовок      — средний шрифт (1.3x), по центру
### Мелкий заголовок — чуть крупнее обычного (1.1x), по центру
> текст              — по центру
---                  — пустая строка
**жирный текст**     — маркеры убираются, текст остаётся
обычный текст        — обычный размер, по левому краю
```

**Ёмкость страницы** (14pt, DPI 200): ~27 строк, ~13 слов/строка, ~350 слов/страницу.

**Пример:**
```markdown
# Линейная алгебра
## Конспект лекции

### Определение
Векторным пространством называется...

> Важно: это фундаментальный результат!

---
```

## Типы раскладки

### Буклет
Листы складываются стопкой и сшиваются посередине. Остаток (< 4 стр.) идёт последовательно.

```
4 стр:  [(1,4),(2,3)]
7 стр:  [(1,4),(2,3),(5,None),(6,7)]
8 стр:  [(1,8),(2,7),(3,6),(4,5)]
9 стр:  [(1,8),(2,7),(3,6),(4,5),(9,None)]
```

### Последовательная
Страницы попарно: 1+2, 3+4, 5+6. Для скоросшивателя/пружины.

### По одной
Каждая страница отдельно по центру A4.

## Установка

```bash
pip install -r requirements.txt
```

## Настройка

Создайте `.env` с ключами API:

```env
BACKEND=gemini          # или openrouter
GEMINI_API_KEY=ваш_ключ
FONT_PATH=/путь/к/шрифту.ttf
MAX_RETRIES=5
```

## Запуск

```bash
python notes_to_docx.py
```

## Зависимости

- Pillow — рендеринг текста в изображения
- python-docx — генерация DOCX
- requests — запросы к AI API
- python-dotenv — загрузка `.env`
